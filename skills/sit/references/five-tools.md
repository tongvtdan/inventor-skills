# SIT — The Five Thinking Tools: Deep Reference

Each tool follows the **Function Follows Form** principle: first modify the product, then discover what the new form is useful for. Do not filter ideas too early — generate freely, evaluate later.

---

## Tool 1: SUBTRACTION

### Core Idea
Remove a component that seems **essential** to the product. Then figure out how to restore the lost function — using only what remains — or discover that the absence creates entirely new value.

The constraint forces elegance. The removed component's absence becomes the innovation driver.

### Step-by-Step Process

1. **List all components** of the product/system
2. **Select one component** — ideally one that seems crucial or central
3. **Virtually remove it** — imagine the product without it
4. Ask: *"What does this modified product look like? What can it still do?"*
5. Ask: *"Can any remaining component perform the removed component's function?"*
6. Ask: *"Who would want this version? What new use case does this open?"*
7. If nothing valuable emerges → try the next component

### When Subtraction Creates Value
- The missing function is provided by the environment or the user instead
- The product becomes simpler, cheaper, more focused, or more portable
- A different user segment values the stripped-down version more than the full version
- The absence itself becomes a differentiator

### Classic Examples

| Product | Removed Component | Innovation |
|---------|-----------------|------------|
| Sony Walkman → Walkman | Recording function | Pocket music player — focused on listening |
| Apple iPod Shuffle | Screen, navigation UI | Ultra-simple, ultra-light music device |
| Laptop → Chromebook | Local storage / OS | Cloud-native, cheap, simple device |
| Camera → Pinhole camera | Lens | Artistic, long-exposure photography |
| Car → Motorcycle | Second pair of wheels + body | Lightweight, agile personal transport |
| Bank → Venmo | Physical branch | Instant peer-to-peer payments |
| Hotel → Airbnb | Hotel-owned properties | Asset-light accommodation network |

### Business Model Subtraction

Remove a step your competitors consider mandatory:
- Remove salespeople → self-serve PLG product
- Remove customer support → AI-first + community-first support
- Remove inventory → dropshipping or print-on-demand
- Remove physical delivery → digital delivery

### Common Mistakes
- Don't remove components at random — prioritize components whose absence could create genuine simplicity or new markets
- Don't discard ideas just because they seem absurd — push through to find the hidden use case

## Tool 2: MULTIPLICATION

### Core Idea
Take an existing component and **copy it** — then alter the copy in some meaningful way (size, shape, location, timing, function). The modified duplicate creates new capabilities or opens new use cases.

Key: the copy must be **different** from the original in at least one important attribute. Simple duplication without modification is not multiplication.

### Step-by-Step Process

1. **Select a component** of the product
2. **Create one or more copies** of it
3. **Modify the copy** — change: size, shape, position, material, timing, function, orientation
4. Ask: *"What does this product do now that it couldn't before?"*
5. Ask: *"What new value does the modified copy add?"*
6. Ask: *"Who benefits from this? What job-to-be-done does it serve?"*

### Dimensions of Modification (pick one or more)
- **Size**: smaller, larger, micro, macro, graduated
- **Position**: moved to a different location on the product
- **Timing**: operates at a different time or frequency
- **Material / substance**: made of a different material
- **Function**: does something different from the original
- **Orientation**: rotated, flipped, angled

### Classic Examples

| Product | Multiplied Component | Modification | Innovation |
|---------|---------------------|--------------|------------|
| Smartphone | Camera | Add: wide, telephoto, macro lenses | Multi-lens system for all photography needs |
| TV | Screen | Add smaller inset screen | Picture-in-picture |
| Razor | Blade | Multiple blades at slightly different angles | Better shave with each blade |
| P&G Febreze | Scent | Alternate between two fragrances | Overcomes nose fatigue; 25% market share gain |
| Keyboard | Key | Add a second Ctrl/Alt on right side | Ergonomic shortcuts |
| Engine | Cylinder | Add more cylinders | Power and smoothness |
| Restaurant | Kitchen | Add satellite cloud kitchen | Ghost kitchen / delivery-only expansion |

### Business Model Multiplication

- Copy a feature and adapt it for a new user segment
- Copy a sales channel and adapt it for a different geography
- Copy a pricing tier and modify one attribute (e.g., usage cap, support level)
- AWS: copied internal infrastructure → modified it as a public service → cloud computing

## Tool 3: DIVISION

### Core Idea
Take a component (or the whole product) and **break it into parts**. Then **rearrange** or **relocate** those parts — either physically, temporally, or functionally — to create new value.

Three types of division:
- **Functional division**: Separate a combined function into independent sub-functions
- **Physical division**: Split a physical component into pieces distributed differently
- **Temporal division**: Separate when a function or component operates

### Step-by-Step Process

1. **Select a component** (or the whole product)
2. **Divide it** — into parts, sub-functions, time slots, or locations
3. **Relocate or resequence** the divided parts
4. Ask: *"What is the divided product capable of now?"*
5. Ask: *"Does distributing the parts solve a problem? Enable something new?"*
6. Ask: *"Who benefits from having the functions separated?"*

### Classic Examples

| Product | Divided Component | Division Type | Innovation |
|---------|-----------------|---------------|------------|
| Television + Remote | Control interface | Functional + Physical | Remote control — separated from the TV body |
| Refrigerator | Cold storage | Physical | Refrigerated drawers under countertop |
| Spirit level (KAPRO) | Bubble vials | Physical | Thin strip with sliding vials for tight spaces |
| Computer | Processing + Display | Physical | Thin client + server-side computing |
| Phone + Camera | Camera function | Functional | Dedicated camera phones then standalone cameras |
| Restaurant | Dining + Kitchen | Physical | Ghost kitchens — kitchen divided from dining experience |
| School | Learning + Credentialing | Functional | MOOCs + certification providers |

### Business Model Division

- Divide your product into a free tier (basic) and paid tier (advanced)
- Divide your team into specialized squads (from generalist pods)
- Divide your go-to-market into product-led (self-serve) + sales-led (enterprise)
- Divide a single license into per-seat, per-usage, per-outcome models

## Tool 4: TASK UNIFICATION

### Core Idea
Take an **external task** that is currently performed outside the product (by the environment, the user, or a separate system) and **assign it to an existing component** inside the product.

Or: take a component that currently does one thing and **give it an additional unrelated task**.

The result: your product does more with what it already has — no new components added.

### Step-by-Step Process

1. **List external tasks** — what tasks happen around your product but not inside it?
2. **Select an external task** — ideally one that requires effort, cost, or creates friction
3. **Identify internal components** that could potentially perform this task
4. Ask: *"If component X performed task Y, what would the product look like?"*
5. Ask: *"What new value does this unification create?"*
6. Ask: *"Does this simplify the user experience? Reduce cost? Create lock-in?"*

Also works in reverse: Take an internal component → assign it a secondary external-facing task.

### Classic Examples

| Original Task | Component That Absorbed It | Unified Innovation |
|--------------|--------------------------|-------------------|
| Navigation + phone + camera | Mobile device | Smartphone |
| Shoe odor control | Sock material | Odor-Eaters socks (deodorizing sock) |
| Crowd data collection | User behavior | Waze — navigation + traffic reporting unified |
| Marketing content | User reviews | TripAdvisor, Yelp — user generates marketing |
| Customer support | Product itself | Self-service UX, in-app guidance, tooltips |
| Payment processing | Messaging app | WeChat Pay, Venmo — payment unified with social |
| Childproofing | Product design | Childproof cap on medicine — safety task unified with packaging |

### Why Task Unification Creates Competitive Advantage

When you absorb a task that customers used to perform externally into your product:
- You reduce friction (fewer tools, fewer steps)
- You create switching costs (customers rely on you for more)
- You capture data from the previously external task
- You eliminate a job that would otherwise need a different product

**BASF case**: Applied Task Unification to chemical plant operations → reassigned 30% of process steps to existing components → reduced operational costs by 30%.

### Business Model Unification

- Unify sales and product (sales process happens inside the product = PLG)
- Unify marketing and product (onboarding = first marketing touchpoint)
- Unify customer success and product (health scores + in-app nudges)
- Unify billing and usage (metered billing = usage tracking + billing unified)

## Tool 5: ATTRIBUTE DEPENDENCY

### Core Idea
Identify two **independent attributes** of a product (or between the product and its environment). Create a **dependency** — so that when one attribute changes, the other automatically changes in response.

This transforms a static product into an **adaptive, responsive, or smart** product — without adding new components.

### Step-by-Step Process

1. **List attributes** — properties of each component: size, color, shape, temperature, speed, price, brightness, transparency, frequency, hardness, quantity, timing...
2. **List external variables** — attributes of the environment or user: time, temperature, light, sound, usage, behavior, location, humidity, age...
3. **Pair an internal attribute with an external variable** (or with another internal attribute)
4. **Create a dependency**: "When [external variable] changes, [internal attribute] changes accordingly"
5. Ask: *"What does this responsive product do now? What problem does it solve?"*
6. Ask: *"Who would pay for this responsive behavior?"*

### Types of Dependencies

- **Product attribute ↔ Environmental attribute**: Product adapts to external conditions
- **Product attribute ↔ User behavior**: Product adapts to how it's used
- **Product attribute ↔ Time**: Product changes over time automatically
- **One product attribute ↔ Another**: Internal coupling creates new emergent function

### Classic Examples

| Attribute 1 (Product) | Attribute 2 (External/Other) | Dependency | Innovation |
|----------------------|------------------------------|------------|------------|
| Lens darkness | UV light level | Darkness ∝ UV | Photochromic / transition glasses |
| Wiper speed | Rainfall intensity | Speed ∝ rain | Auto rain-sensing wipers |
| Bottle color | Liquid temperature | Color changes at threshold | Baby bottle temperature indicator |
| Drug release rate | Body conditions | Rate ∝ need | Smart drug delivery systems |
| Pricing | Demand level | Price ∝ demand | Uber surge pricing |
| Content shown | User behavior | Content ∝ behavior | Netflix / TikTok recommendations |
| Car handling | Road conditions | Stiffness ∝ road | Active suspension systems |
| Coffee strength | Time of day | Strength ∝ preference | Smart coffee machine profiles |

### Business Model Attribute Dependency

- Revenue ↔ Customer success (success-based pricing)
- Feature access ↔ Usage level (freemium thresholds)
- Price ↔ Volume purchased (tiered pricing)
- Support priority ↔ Contract tier (SLA tiers)
- Product behavior ↔ User maturity (progressive disclosure in UX)

### Why This Tool Is Powerful

Attribute Dependency creates **personalization at zero marginal cost** — the product adapts without human intervention. Every smart, adaptive, or context-aware product is an example of engineered attribute dependencies.

## Combining the Five Tools

SIT tools are not mutually exclusive. The most powerful innovations often combine multiple tools:

| Combination | Effect | Example |
|-------------|--------|---------|
| Subtraction + Task Unification | Remove a component; give its job to another | GPS inside phone: dedicated GPS device subtracted, function given to phone chip |
| Multiplication + Attribute Dependency | Duplicate a component; link the copies' behavior to context | Multi-camera phone: which camera activates depends on zoom level |
| Division + Task Unification | Break into pieces; each piece also does a second job | Modular laptop: battery compartment = wireless charger for phone |
| Subtraction + Division | Remove the whole; use only the parts | Remote control: TV body removed from hands, only control interface remains |

## Facilitation Tips for SIT Sessions

- **Time**: 30–90 minutes is typical for a SIT ideation session
- **Group**: 3–8 people; diversity of perspective helps
- **Rules**: No immediate evaluation during generation phase — "Yes, and..." not "But..."
- **Stimulus**: Start with the component list on a whiteboard; physically remove/add/link components as you work
- **Output**: Capture all virtual products, even absurd ones — evaluate in a separate phase
- **Follow-up**: Top 2–4 concepts go to rapid prototyping or user interviews within 48 hours
