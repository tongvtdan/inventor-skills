# Contributing to inventor-skills

Thank you for helping make systematic invention more accessible. Contributions are welcome across:

- New TRIZ application examples (engineering, software, business)
- Additional SIT tool walkthroughs
- Architecture / domain-specific applications (e.g., TRIZ for architecture, SIT for UX)
- Bug fixes in methodology descriptions
- Better trigger phrases in skill descriptions

## How to Contribute

1. Fork the repository
2. Create a branch: `git checkout -b feat/your-addition`
3. Make your changes (see guidelines below)
4. Submit a Pull Request with a clear description

## File Guidelines

- Keep `SKILL.md` files under 3,000 words
- Put detailed reference content in the `references/` folder
- Use plain, imperative language (Claude reads these as instructions)
- Test your skill by triggering it in Claude and reviewing the output
- Add real-world examples — not hypothetical ones

## What Makes a Good Addition

- Specific: "Add the TRIZ/SIT application to [domain X]" not "improve TRIZ"
- Grounded: based on documented real-world use cases
- Testable: include 1–2 example prompts that should trigger it
